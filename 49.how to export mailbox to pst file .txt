
49. How to Export Mailbox to PST File in Exchange Server 2019

### **Exchange 2019 Mailbox Export to PST Guide**

This guide provides a structured, step-by-step technical walkthrough for exporting a user mailbox to a PST file using the Exchange Admin Center (EAC), based on the provided source.

---

### **I. Step 1: Assigning Export Permissions**
By default, even the Administrator account does not have the permissions required to export mailboxes. You must manually assign the **Mailbox Import Export** role.

1.  **Access EAC:** Log in to the Exchange Admin Center.
2.  **Navigate:** Go to **Permissions** and select **Admin Roles**.
3.  **Add New Role:** Click the **plus sign (+)** to create a new role group (e.g., "Mailbox Management").
4.  **Add Role:** In the Roles section, click **(+)** and select **Mailbox Import Export** from the list.
5.  **Add Member:** In the Members section, click **(+)** to add your administrator account (e.g., "Administrator").
6.  **Save:** Click **Save**. You may need to log out and log back in for the permission change to take effect.

---

### **II. Step 2: Creating a Shared Network Folder**
Exchange requires a **UNC path** (network share) to save the PST file. The export will fail if you attempt to use a local drive path (like `C:\PST`).

1.  **Create Folder:** Create a folder on the server (e.g., `C:\PSTExport`).
2.  **Share Folder:** Right-click the folder, go to **Properties > Sharing**, and click **Advanced Sharing**.
3.  **Permissions:** Click **Permissions** and then **Add**.
4.  **Exchange Trusted Subsystem:** You must add the **Exchange Trusted Subsystem** group and grant it **Full Control** (Read/Change).
    *   *Note: According to Microsoft, this group is essential for the Exchange server to write the file to the shared folder.*
5.  **Identify UNC Path:** Note the network path (e.g., `\\ServerName\PSTExport`).

---

### **III. Step 3: Initiating the Export Process**
1.  **Navigate:** In the EAC, go to **Recipients** and click the **Mailboxes** tab.
2.  **Select Mailbox:** Choose the mailbox you want to export (e.g., "Jaisa").
3.  **Export Tool:** Click the **three dots (...)** (More) and select **Export to PST**.
    *   *Note: If you do not see this option, verify your permissions from Step 1.*
4.  **Scope:** Select the radio button to "Export only the contents of this mailbox" (standard) and click **Next**.

---

### **IV. Step 4: Specifying the Target Path**
1.  **UNC Path:** Enter the full network path including the filename and extension (e.g., `\\ServerName\PSTExport\jaisa.pst`).
2.  **Notification:** Select a mailbox to receive an automated notification email once the export is finished successfully.
3.  **Finish:** Click **Finish**. The export runs as a background process.

---

### **V. Step 5: Verification and Completion**
1.  **Check Shared Folder:** Navigate to your `PSTExport` folder to confirm the `.pst` file has been generated.
2.  **Monitor Email:** Log in to **Outlook Web App (OWA)** as the administrator or the user designated in Step 4.
3.  **Status Mail:** You should receive an email with the subject "The mailbox export request has been received" followed by a confirmation once it has **successfully exported**.

---

**Summary Tip:** This process is reversible; you can use the same shared folder and the "Import PST" option under the mailbox settings (three dots menu) to restore data if needed.
