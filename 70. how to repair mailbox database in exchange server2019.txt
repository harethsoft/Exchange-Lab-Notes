70. How to Repair a Mailbox Database in Exchange Server 2019


### **Exchange 2019 Mailbox Database Repair Guide**

This guide provides a structured, step-by-step technical walkthrough for repairing a corrupted mailbox database in Exchange Server 2019 using the **ESEUTIL** command-line tool, based on the provided source.

---

### **I. Initial Assessment & Preparation**
When a mailbox database fails to mount, it is often due to corruption or missing transaction logs. 

1.  **Safety First:** Before performing any repair, **create a backup or a copy** of the database (`.edb`) file and its associated log files.
2.  **Locate Database Files:** Identify the physical path of the database and its log files. For example, a common path might be `C:\Program Files\Microsoft\Exchange Server\V15\Mailbox\DatabaseName`.
3.  **Open Command Prompt:** Run the Command Prompt as an **Administrator** and navigate to the directory containing your database files.

---

### **II. Checking the Database State**
The first step in troubleshooting is determining if the database is in a **Clean Shutdown** or **Dirty Shutdown** state.

1.  **Run Health Check:** Use the following command:
    *   `eseutil /mh "Path_To_Database.edb"`.
2.  **Analyze the "State" Field:**
    *   **Clean Shutdown:** All transactions have been committed to the database. It should mount successfully.
    *   **Dirty Shutdown:** Transactions are missing from the database, or log files are corrupted. The database **cannot be mounted** in this state.
3.  **Identify Required Logs:** If in a "Dirty Shutdown" state, the output will list the "Log Required" range (e.g., from log 17 to 36).

---

### **III. Performing Soft Recovery (/R)**
A **Soft Recovery** attempts to replay the missing transaction logs into the database to bring it back to a clean state.

1.  **Command Syntax:**
    *   `eseutil /r <LogPrefix> /d`.
    *   *Example:* If your log files start with "E01", the command would be `eseutil /r E01 /d`.
2.  **Execution:** The tool will attempt to re-link the database with its logs. If the operation is successful, it will display **"Operation completed successfully"**.

---

### **IV. Performing Hard Repair (/P)**
If a Soft Recovery fails (usually because log files are missing or too badly corrupted), a **Hard Repair** is required.

1.  **Command Syntax:**
    *   `eseutil /p "Path_To_Database.edb"`.
2.  **Warning:** A Hard Repair **discards any corrupted data** it cannot fix. This may result in **data loss** for the items contained within those corrupted sectors.
3.  **Execution:** Run the command and wait for the repair process to finalize.

---

### **V. Final Verification & Mounting**
After performing a repair, you must verify the database is now in a healthy state before attempting to mount it.

1.  **Final Health Check:** Run the health check command again:
    *   `eseutil /mh "Path_To_Database.edb"`.
2.  **Verify State:** The "State" field should now read **Clean Shutdown**.
3.  **Mount the Database:**
    *   Open the **Exchange Admin Center (EAC)**.
    *   Go to **Servers > Databases**.
    *   Select your database, click the **three dots (...)**, and select **Mount**.
4.  **Confirm Status:** The database status should now show as **Mounted**.
