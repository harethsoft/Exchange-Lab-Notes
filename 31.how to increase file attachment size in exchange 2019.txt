
## 31. Increase File Attachment Size in Exchange Server 2019

### **Exchange 2019 File Attachment Size Configuration Guide**

This guide provides a structured, step-by-step technical walkthrough for increasing the file attachment size limit in Exchange Server 2019, based on the provided source.

---

### **I. Understanding Message vs. Attachment Limits**
*   **Message Size Limit:** The total size of a message, including the header, body, and all attachments.
*   **Attachment Size Limit:** Specifically applies to individual file attachments. 
*   **Default Limit:** In a standard Exchange 2019 installation, the default maximum size for attachments is **10 MB**.

---

### **II. Checking Current Limits (Exchange Management Shell)**
Before making changes, use the Exchange Management Shell (EMS) to identify where the current limits are applied.

1.  **Transport Configuration:** Check the organization-wide limits.
    *   Get-TransportConfig | Format-List MaxSendSize, MaxReceiveSize
2.  **Receive Connectors:** Check individual connector limits (often default to 36 MB).
    *   Get-ReceiveConnector | Select-Object Name, MaxMessageSize
3.  **Send Connectors:** Check limits on outgoing connectors (if created).
    *   Get-SendConnector | Format-List Name, MaxMessageSize
4.  **User Mailbox:** Check if a specific user has custom limits.
    *   Get-Mailbox -Identity "it1" | Format-List Name, MaxSendSize, MaxReceiveSize

---

### **III. Increasing Limits via Exchange Admin Center (EAC)**
The most effective way to increase the attachment limit across the organization is by modifying the **Organization Transport Settings**.

1.  **Access EAC:** Log in to the Exchange Admin Center (e.g., `https://localhost/ecp`).
2.  **Navigate:** Go to **Mail flow** and select the **Receive connectors** tab.
3.  **Transport Settings:** Click the **three dots (...)** for more options and select **Organization transport settings**.
4.  **Modify Limits:** Under the **Limits** tab, update the following fields:
    *   **Maximum received message size (MB):** Change from 10 to your desired value (e.g., **20 MB**).
    *   **Maximum sent message size (MB):** Change from 10 to your desired value (e.g., **20 MB**).
5.  **Save:** Click **Save** to apply the new values.

---

### **IV. Applying Changes (Service Restart)**
Changes to transport settings require a restart of Exchange services to take effect.

1.  **Verify via EMS:** Run `Get-TransportConfig | Format-List MaxSendSize, MaxReceiveSize` to confirm the values have updated in the database.
2.  **Restart Services:** In a test environment, you can restart all Exchange-related services via PowerShell:
    *   `Get-Service | Where-Object { $_.DisplayName -notlike "*Hyper-V*" } | Restart-Service -Force`.
3.  **Production Note:** In a production environment, you should **schedule** the restart of the Exchange Server to avoid interrupting active mail flow.

---

### **V. Final Verification**
1.  **Log in to OWA:** Access Outlook on the Web as a user.
2.  **Compose Mail:** Create a new message and click the **Attachment** icon.
3.  **Test Upload:** Select a file that exceeds the previous 10 MB limit but is within the new 20 MB limit (e.g., a **17.5 MB** PDF).
4.  **Success:** The file should now attach successfully without the "maximum size limit" error message.
